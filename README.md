## 架构概述

```
PL/0 源代码
    │
    ▼
┌──────────┐    ┌──────────┐    ┌──────────┐
│ 词法分析  │ → │ 语法分析  │ → │ 语义分析  │
│ lexer.c  │    │lr_parser.c│    │semantic.c│
└──────────┘    └──────────┘    └──────────┘
  Token 流        语法正确/错误    符号表 + 四元式
```

三个模块**完全独立**，各自重新初始化词法分析器，互不共享解析状态。

## 词法规则

### Token 分类

| 类别 | 元素 |
|------|------|
| 保留字 | `const` `var` `procedure` `if` `then` `while` `do` `call` `read` `write` `begin` `end` `odd` |
| 运算符 | `+` `-` `*` `/` `:=` `=` `#` `<` `<=` `>` `>=` |
| 界符 | `,` `;` `.` `(` `)` |
| 标识符 | 字母或下划线开头，字母/数字/下划线组成，**长度 ≤ 8**，**大小写不敏感** |
| 无符号整数 | 数字组成，**长度 ≤ 8** |

### 注释

| 格式 | 示例 |
|------|------|
| 单行 | `// 注释内容` |
| 多行 (C风格) | `/* 多行注释 */` |
| 多行 (Pascal风格) | `(* 多行注释 *)` |

### 词法错误类型

| 错误类型 | 输出格式 | 触发条件 |
|----------|----------|----------|
| 非法字符(串) | `(非法字符(串),X,行号:N)` | 出现 `@` `&` `!` 等未定义字符；或数字开头后跟字母的非法标识符（如 `2a`） |
| 无符号整数越界 | `(无符号整数越界,N,行号:N)` | 无符号整数长度超过 8 位 |
| 标识符长度超长 | `(标识符长度超长,name,行号:N)` | 标识符长度超过 8 个字符 |

## 语法规则（EBNF）

```
<程序>             ::= <分程序> .
<分程序>           ::= [<常量说明部分>] [<变量说明部分>] [<过程说明部分>] <语句>
<常量说明部分>     ::= const <常量定义> { , <常量定义> } ;
<常量定义>         ::= <标识符> = <无符号整数>
<变量说明部分>     ::= var <标识符> { , <标识符> } ;
<过程说明部分>     ::= <过程首部> <分程序> { ; <过程说明部分> } ;
<过程首部>         ::= procedure <标识符> ;
<语句>             ::= <赋值语句> | <条件语句> | <当型循环语句>
                     | <过程调用语句> | <读语句> | <写语句> | <复合语句> | <空语句>
<赋值语句>         ::= <标识符> := <表达式>
<条件语句>         ::= if <条件> then <语句>
<当型循环语句>     ::= while <条件> do <语句>
<过程调用语句>     ::= call <标识符>
<读语句>           ::= read ( <标识符> { , <标识符> } )
<写语句>           ::= write ( <表达式> { , <表达式> } )
<复合语句>         ::= begin <语句> { ; <语句> } end
<条件>             ::= <表达式> <关系运算符> <表达式> | odd <表达式>
<表达式>           ::= [ + | - ] <项> { <加减运算符> <项> }
<项>               ::= <因子> { <乘除运算符> <因子> }
<因子>             ::= <标识符> | <无符号整数> | ( <表达式> )
<关系运算符>       ::= = | # | < | <= | > | >=
<加减运算符>       ::= + | -
<乘除运算符>       ::= * | /
```

## 错误类型说明

| 错误类别 | 输出格式 | 含义 |
|----------|----------|------|
| 词法-非法字符(串) | `(非法字符(串),X,行号:N)` | 出现 PL/0 不接受的字形 |
| 词法-整数越界 | `(无符号整数越界,N,行号:N)` | 整数超过 8 位 |
| 词法-标识符超长 | `(标识符长度超长,name,行号:N)` | 标识符超过 8 字符 |
| 语法错误 | `(语法错误,行号:N)` | Token 序列不满足文法，缺少必要符号或出现意外 Token |
| 语义-重复声明 | `(语义错误,行号:N)` | 同一作用域内标识符被重复定义 |
| 语义-未声明 | `(语义错误,行号:N)` | 使用未声明的标识符 |
| 语义-类型不匹配 | `(语义错误,行号:N)` | 操作要求变量却给了过程，反之亦然 |

## 实验四：词法分析测试用例

### e4_t1: 正确词法测试

**源文件**：`tests/experiment4_lexer/e4_t1_correct.pl0`

```
const a = 10;
var   b, c;
procedure fun1;
    if a <= 10 then
        begin
            c := b + a;
        end;
begin
    read(b);
    while b # 0 do
        begin
            call fun1;
            write(2 * c);
            read(b);
        end
end.
```

**分析**：此程序是语法和语义均正确的 PL/0 程序，所有 54 个 Token 均应正常识别，无任何词法错误。

---

### e4_t2: 错误词法测试（非法字符/数字越界/标识符过长）

**源文件**：`tests/experiment4_lexer/e4_t2_error.pl0`

```
const 2a = 123456789;
var   b, c;

//单行注释

/*
* 多行注释
*/

procedure function1;
    if 2a <= 10 then
        begin
            c := b + a;
        end;
begin
    read(b);
    while b @ 0 do
        begin
            call function1;
            write(2 * c);
            read(b);
        end
end.
```

**逐错误分析**：

| 行号 | 源代码 | 错误输出 | 原因分析 |
|------|--------|----------|----------|
| 1 | `const 2a = ...` | `(非法字符(串),2a,行号:1)` | `2a` 以数字开头后跟字母。词法分析器读到数字 `2` 后继续读 `a`，发现这是非法标识符（数字开头 + 字母组合），不属于 PL/0 的合法 Token |
| 1 | `... = 123456789;` | `(无符号整数越界,123456789,行号:1)` | `123456789` 是 9 位数字，超过 PL/0 规定的**无符号整数最大 8 位**限制（`MAX_NUM_LEN = 8`） |
| 4-8 | 注释块 | — | 两种注释（`//` 和 `/* */`）均被正确跳过，不产生任何 Token，不报错 |
| 10 | `procedure function1;` | `(标识符长度超长,function1,行号:10)` | `function1` 共 9 个字符，超过 PL/0 规定的**标识符最大 8 位**限制（`MAX_IDENT_LEN = 8`） |
| 11 | `if 2a <= 10 then` | `(非法字符(串),2a,行号:11)` | 同第 1 行：`2a` 是数字开头的非法标识符 |
| 17 | `while b @ 0 do` | `(非法字符(串),@,行号:17)` | `@` 不在 PL/0 字符集中。PL/0 的关系运算符应为 `=` `#` `<` `<=` `>` `>=`，`@` 是非法字符 |
| 19 | `call function1;` | `(标识符长度超长,function1,行号:19)` | 同第 10 行：`function1` 标识符长度超限 |

> 注释（行 4-8）虽然被词法分析器正确跳过，但在行号统计中占用了行号空间，因此 `function1` 出现在行 10 而非行 4。

### e5_t5: 多个语法错误（8 个错误）

**源文件**：`tests/experiment5_parser/e5_t5_multi_error.pl0`

```
const a := 10;
var   b, c d;

//单行注释

/*
* 多行注释
*/

procedure procedure fun1;
    if a <= 10
        begin
            c = b + a
        end;
begin
    read(b;
    while b # 0
        begin
            call fun1;
            write 2 * c);
            read(b);
        end
end.
```

**逐错误分析**：

| 行号 | 源代码 | 错误输出 | 原因分析 |
|------|--------|----------|----------|
| 1 | `const a := 10;` | `(语法错误,行号:1)` | `<常量定义>` 应为 `=` 而非赋值符号 `:=`。同 e5_t2 |
| 2 | `var b, c d;` | `(语法错误,行号:2)` | 变量声明中 `c` 和 `d` 之间缺少逗号 `,`。解析器匹配 `c` 后期望 `,` 或 `;`，但遇到 `d`（仍为标识符），说明此处缺少界符分隔 |
| 10 | `procedure procedure fun1;` | `(语法错误,行号:10)` | `<过程首部>` 要求 `procedure <标识符> ;`，但此处出现了连续的 `procedure procedure`。第一个 `procedure` 被视为进入过程声明，第二个 `procedure` 被当作 `<标识符>` 期望的位置，但 `procedure` 是关键字不是标识符 |
| 11 | `if a <= 10` (下一行 `begin` 而非 `then`) | `(语法错误,行号:11)` | `<条件语句>` 要求 `if <条件> then <语句>`。`a <= 10` 作为条件被成功解析，但下一 Token 是 `begin` 而非 `then`——缺少 `then` 关键字 |
| 13 | `c = b + a` | `(语法错误,行号:13)` | `<赋值语句>` 必须使用 `:=` 而非 `=`。`=` 是关系运算符（等于比较），不能用于赋值 |
| 16 | `read(b;` | `(语法错误,行号:16)` | `<读语句>` 要求 `read ( <标识符> )`，此处缺少右括号 `)` |
| 17 | `while b # 0` (下一行 `begin` 而非 `do`) | `(语法错误,行号:17)` | 同 e5_t4：`while` 循环缺少 `do` 关键字 |
| 20 | `write 2 * c);` | `(语法错误,行号:20)` | `<写语句>` 要求 `write ( <表达式> )`，此处缺少左括号 `(`。解析器在 `write` 后期望 `(`，但遇到的是数字 `2` |

### e6_t2: 重复声明

**源文件**：`tests/experiment6_semantic/e6_t2_dup_declare.pl0`

```
const a = 10;
var   a, b, c;
...
```

**逐错误分析**：

| 行号 | 源代码 | 错误输出 | 原因分析 |
|------|--------|----------|----------|
| 2 | `var a, b, c;` | `(语义错误,行号:2)` | `a` 已在第 1 行被声明为常量 `const a = 10`。PL/0 不允许同一作用域内标识符被重复声明。虽然 const 和 var 是不同的符号类型，但 `add_symbol()` 检查的是名称唯一性（跨类型），因此 `a` 被标记为重复声明 |

## 输出格式说明

### 词法分析输出

每个 Token 按类别输出：

```
(保留字,const)         — 关键字
(标识符,a)             — 标识符
(运算符,:=)            — 运算符
(无符号整数,10)        — 整数
(界符,;)              — 界符
```

遇到词法错误时，错误信息**内联**在 Token 流对应位置：

```
(非法字符(串),2a,行号:1)              — 非法字形
(无符号整数越界,123456789,行号:1)      — 数字超 8 位
(标识符长度超长,function1,行号:10)    — 标识符超 8 字符
```

末尾输出各类 Token 的统计计数。

### 语法分析输出

```
递归下降语法分析过程
--------------------
语法正确                  — 成功

(语法错误,行号:N)          — 失败（每个语法不匹配点输出一行）
语法分析发现错误。
```

若存在词法错误，语法和语义阶段会被跳过。

### 语义分析输出

```
✅ 语义分析成功！

========== 符号表 ==========
const a = 10
var b
var c
procedure p

========== 中间代码(四元式) ==========
(1)(syss,_,_,_)
(2)(const,a,_,_)
...
```

若存在语义错误，仅输出错误信息，不生成四元式。
